# 可访问性测试清单

本文件提供了手动测试可访问性的详细步骤。

---

## ♿ 键盘导航测试

### 1. Skip Link 测试

**步骤**：
1. 刷新页面
2. 按 `Tab` 一次
3. ✅ 应该看到 "跳转到主内容" 链接
4. 按 `Enter`
5. ✅ 焦点应该移动到 `<main>` 元素
6. ✅ 链接应该隐藏（除非有焦点）

### 2. 导航菜单测试

**步骤**：
1. 导航到任何页面
2. 按 `Tab` 浏览顶部导航
3. ✅ 所有链接应该可以通过 `Enter` 激活
4. ✅ 当前页面应该有 `aria-current="page"`
5. ✅ 焦点指示器清晰可见

**移动端菜单**：
1. 调整浏览器宽度到 <768px
2. 点击菜单按钮（或使用 Tab 导航到它）
3. ✅ `aria-expanded` 应该更新
4. ✅ 菜单应该正确显示/隐藏
5. ✅ `Escape` 应该关闭菜单

### 3. 图片卡片测试

**步骤**：
1. 导航到图片管理页面
2. 使用 `Tab` 导航到图片卡片
3. ✅ 每个卡片应该有 `role="button"`
4. ✅ 按 `Enter` 或 `Space` 应该打开预览
5. ✅ 在选择模式，应该切换选中状态
6. ✅ 焦点应该有清晰的环

### 4. 上传区域测试

**步骤**：
1. 导航到首页
2. 使用 `Tab` 导航到上传区域
3. ✅ 应该可以 `Enter` 或 `Space` 打开文件选择
4. ✅ `aria-label` 应该描述区域用途

### 5. 上传队列测试

**步骤**：
1. 上传一些文件
2. 使用 `Tab` 导航上传队列
3. ✅ 进度条应该有 `role="progressbar"`
4. ✅ 应该有 `aria-valuenow`, `aria-valuemin`, `aria-valuemax`
5. ✅ 删除按钮应该有 `aria-label`
6. ✅ 状态图标应该有 `aria-label`

### 6. 按钮测试

**步骤**：
1. 浏览所有页面
2. 使用 `Tab` 导航所有按钮
3. ✅ 所有按钮应该可以通过键盘激活
4. ✅ 图标按钮必须有 `aria-label`
5. ✅ 禁用按钮应该被跳过

## 🎨 对比度测试

### 工具

使用以下工具之一：
- **WebAIM Contrast Checker**: https://webaim.org/resources/contrastchecker/
- **axe DevTools** 浏览器扩展
- **WAVE** 浏览器扩展
- **Lighthouse**

### 检查项目

#### 浅色模式

- [ ] 正文文本 ≥ 4.5:1
- [ ] 标题文本 ≥ 4.5:1
- [ ] 辅助文本 ≥ 4.5:1
- [ ] 图标 ≥ 3:1
- [ ] 按钮文本 ≥ 4.5:1

#### 深色模式

- [ ] 正文文本 ≥ 4.5:1
- [ ] 标题文本 ≥ 4.5:1
- [ ] 辅助文本 ≥ 3:1（大文本）
- [ ] 图标 ≥ 3:1
- [ ] 边框和分割线 ≥ 3:1

### 测试步骤

1. 打开页面
2. 使用颜色选择器工具
3. 检查前景色和背景色
4. 记录对比度比例
5. 切换深色模式重复检查

## 🖱️ 屏幕阅读器测试

### VoiceOver (macOS)

**启用**：
- `Cmd + F5`

**测试**：
1. 启用 VoiceOver
2. 使用 `VO + Right Arrow` 导航
3. 检查所有元素是否有描述
4. 检查焦点是否按预期移动

### NVDA (Windows)

**启用**：
- `Ctrl + Alt + N`

**测试**：
1. 启用 NVDA
2. 使用 `Tab` 导航
3. 检查是否所有交互元素被正确朗读

### ChromeVox (Chrome)

**启用**：
- 安装 ChromeVox 扩展
- `Ctrl + Alt + Z`

## 🎬 动画测试

### 减少运动偏好

**测试步骤**：
1. 系统设置 → 辅助功能 → 减少运动
2. 启用 "减少运动"
3. 刷新页面
4. ✅ 动画应该停止或大幅减少
5. ✅ 内容应该仍然可访问

**快速测试（开发环境）**：
1. 打开 DevTools
2. 运行 `matchMedia('(prefers-reduced-motion: reduce)').matches`
3. 应该返回 `true` 或 `false`

### 动画时长

使用浏览器 DevTools Performance 面板：

1. 录制交互
2. 检查动画持续时间
3. ✅ 应该在 150-300ms 范围内

## 📱 触摸目标测试

### 最小尺寸

**工具**：
- Chrome DevTools → Elements → 检查元素 → Computed → 查看尺寸

**检查项目**：
- [ ] 所有按钮 ≥ 44×44px
- [ ] 图标按钮 ≥ 44×44px（通过 padding）
- [ ] 链接 ≥ 44×44px
- [ ] 输入框 ≥ 44px 高度

### 间距

**检查项目**：
- [ ] 触摸目标之间 ≥ 8px
- [ ] 没有元素太接近

# 🔄 配置自动同步优化

## 问题描述

**用户反馈：** "刷新页面之后应该先获取配置啊，如果登录状态的话。"

### 原有行为 ❌

`ConfigDiscovery` 组件只在以下条件**同时满足**时运行：
```typescript
enabled: !!session?.accessToken && !configStore.owner
//           已登录 ✅              且未配置 ✅
```

**问题场景：**
1. 用户在设备 A 配置了图床 → `configStore.owner` 有值
2. 用户在其他地方修改了 GitHub 上的配置
3. 用户刷新页面或换设备登录
4. localStorage 恢复了旧配置 → `configStore.owner` 有值
5. ❌ **ConfigDiscovery 不运行** → 不会从 GitHub 获取最新配置
6. 导致本地配置可能与远程配置不一致

---

## 解决方案 ✅

### 修改后的行为

**新的触发条件：**
```typescript
enabled: !!session?.accessToken
//        已登录即可 ✅
```

**新增的缓存策略：**
```typescript
staleTime: 60 * 1000,        // 1 分钟内不重复请求
gcTime: 5 * 60 * 1000,       // 5 分钟后清理缓存
```

### 工作流程

#### 场景 1：已登录 + 未配置
```
用户刷新页面
  ↓
ConfigDiscovery 运行 ✅
  ↓
从 GitHub 发现配置
  ↓
更新 configStore
  ↓
用户可以正常使用
```

#### 场景 2：已登录 + 已配置
```
用户刷新页面
  ↓
localStorage 恢复配置
  ↓
ConfigDiscovery 运行 ✅（新逻辑）
  ↓
从 GitHub 获取最新配置
  ↓
对比并更新 configStore
  ↓
用户使用最新配置
```

#### 场景 3：未登录
```
用户刷新页面
  ↓
ConfigDiscovery 不运行 ❌（正确行为）
  ↓
显示登录引导
```

## 技术实现

### 修改文件
**`src/hooks/useConfigDiscovery.ts`**

### 核心改动

#### 1. 触发条件

**修改前：**
```typescript
enabled: !!session?.accessToken && !configStore.owner,
staleTime: Infinity,
gcTime: Infinity,
```

**修改后：**
```typescript
enabled: !!session?.accessToken,
staleTime: 60 * 1000,  // 1 分钟缓存
gcTime: 5 * 60 * 1000, // 5 分钟 GC
```

#### 2. 代码注释

添加了更详细的注释说明触发条件：
```typescript
/**
 * Hook: 发现用户 GitHub 仓库中的 ImgX 配置
 * 使用固定仓库名: img.shenzjd.com
 *
 * 触发条件：
 * 1. 已登录且未配置 → 发现并加载配置
 * 2. 已登录且已配置 → 从 GitHub 同步最新配置
 */
```

## 注意事项

### ⚠️ GitHub API 限流
- GitHub API 有请求频率限制（未认证 60 次/小时，认证 5000 次/小时）
- `staleTime: 60 秒` 确保不会频繁请求
- 普通用户使用完全够用

### ⚠️ 配置冲突
- 如果本地配置和远程配置不同，以**远程配置**为准
- 修改配置后建议点击"立即同步配置到 GitHub"按钮

### ⚠️ 网络错误
- 如果 GitHub API 请求失败（网络问题），不影响本地配置
- 会静默失败，下次刷新时重试

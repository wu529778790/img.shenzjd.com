### 修复退出登录 Bug
- **问题**: 点击"退出登录"后没有真正退出，只是跳转回首页，账号信息仍然显示
- **根因**: `src/app/api/auth/signout/route.ts` 自定义路由覆盖了 NextAuth v4 内置的 signout 处理器，但该自定义路由只读取 session 并返回 `{success: true}`，从未清除 session cookie
- **修复**:
  1. 删除 `src/app/api/auth/signout/route.ts`（让 NextAuth 内置 signout 处理器正常工作）
  2. 优化 `Header.tsx` 的 `handleLogout`: 使用 `signOut({ redirect: false })` + `window.location.href = '/'` 替代 `signOut({ callbackUrl: '/' })`，确保 cookie 先被清除再硬跳转
  3. 添加 `queryClient.clear()` 清除 React Query 缓存

### 修复骨架屏加载闪烁
- **问题**: 骨架屏外层有 `<div className="min-h-[60vh]">` 包裹，与实际内容的外层结构 (`min-h-0`) 不一致，导致切换时产生宽度/高度跳变闪烁
- **修复**: 移除 management/page.tsx 中骨架屏的多余 wrapper，让 `<ManagementSkeleton />` 直接作为返回值，其内部 `min-h-0` → `container max-w-5xl` 结构与实际页面完全一致
- **问题**: 首页上传图片后，切换到图片管理页看不到新上传的图片，需手动刷新
- **根因**: `useImages.ts` 中 `staleTime: 2 * 60 * 1000`（2 分钟），数据在缓存期内即使被 `invalidateQueries` 标记，组件重新挂载时 React Query 也可能服务旧缓存
- **修复**: `staleTime` 改为 `0`，确保每次进入管理页都重新获取最新数据

### 修复上传后图片管理页看不到新图片
- **根因**: `Header.tsx` 中 `prefetchManagementPage` 设了 `staleTime: 60000`（1分钟），鼠标悬停预取后缓存带 60s 新鲜期；用户点进管理页时，`useImages` 虽设了 `staleTime: 0`，但 React Query 沿用预取时的 staleTime，导致 60s 内都直接服务旧缓存
- **修复**: `prefetchManagementPage` 的 `staleTime` 改为 `0`，与 `useImages` 保持一致

### 技术实现

#### 1. Git Trees API（1次请求）

```typescript
GET /repos/{owner}/{repo}/git/trees/{branch}?recursive=1

// 返回所有文件和文件夹
{
  "tree": [
    {
      "type": "blob",
      "path": "images/photo.jpg",
      "sha": "abc123",
      "size": 12345
    }
  ]
}
```

#### 2. 排序逻辑

```typescript
// 按文件名排序
files.sort((a, b) => a.name.localeCompare(b.name))

// 按文件大小排序
files.sort((a, b) => a.size - b.size)

// 按文件路径排序
files.sort((a, b) => a.path.localeCompare(b.path))
```

## 📊 完整优化历程

### 第一阶段：Git Trees API + 缓存（已废弃）

- 文件列表：N次 → 1次
- 提交时间：首次N次，缓存后0次
- **总请求数**：首次 1+N次，缓存后 1次

### 第二阶段：GraphQL API（失败）

- 尝试使用 GraphQL API 批量获取提交时间
- 发现 GitHub GraphQL API 不支持此用法
- ❌ `entries` 字段不接受 `names` 参数
- ❌ `Blob` 类型没有 `commitHistory` 字段

### 第三阶段：方案 C - 其他字段排序（最终）⭐

- **文件列表：1次**
- **提交时间：0次**
- **总请求数：1次**（稳定）

## 🔄 排序功能

### 可用的排序字段

| 排序字段 | 说明 | 示例 |
|---------|------|------|
| **名称** | 按文件名字母顺序 | photo.jpg < screenshot.png |
| **大小** | 按文件大小（字节） | 1KB < 10KB < 1MB |
| **路径** | 按完整路径字母顺序 | images/2024/01/ < images/2024/02/ |

### 排序方向

- **升序（asc）**：A → Z，小 → 大
- **降序（desc）**：Z → A，大 → 小

### 默认排序

- **字段**：名称（name）
- **方向**：升序（asc）

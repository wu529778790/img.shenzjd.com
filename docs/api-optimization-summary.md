# 图片管理 API 优化完成

## 优化内容

### ✅ 1. 使用 Git Trees API 替代递归遍历文件夹

**修改文件**：`src/lib/github.ts:68-101`

**优化前**：
- 每个文件夹触发一次 `GET /repos/{owner}/{repo}/contents/{path}` 请求
- 10层文件夹 = 10次请求

**优化后**：
- 使用 `GET /repos/{owner}/{repo}/git/trees/{branch}?recursive=1`
- 无论多少层文件夹 = **1次请求**
- 性能提升：**10倍 - 100倍**（取决于文件夹深度）

**回退机制**：
- 如果 Git Trees API 失败，自动回退到原来的递归方式
- 确保兼容性

---

### ✅ 2. 添加提交时间缓存机制

**修改文件**：`src/lib/github.ts:301-375`

**优化前**：
- 每次刷新页面都重新请求所有文件的提交时间
- 100张图片 = 100次请求

**优化后**：
- 内存缓存提交时间（5分钟有效期）
- 首次加载后，5分钟内再次访问直接从缓存读取
- 大幅减少重复请求

**缓存策略**：
- 类型：内存缓存（Map）
- 有效期：5分钟
- 自动过期清理

## 技术细节

### 1. Git Trees API

**请求示例**：
```bash
GET https://api.github.com/repos/{owner}/{repo}/git/trees/{branch}?recursive=1
```

**响应示例**：
```json
{
  "tree": [
    {
      "type": "blob",
      "path": "images/photo.jpg",
      "sha": "abc123",
      "size": 12345
    },
    {
      "type": "blob",
      "path": "images/2024/01/photo2.jpg",
      "sha": "def456",
      "size": 23456
    },
    {
      "type": "tree",
      "path": "images",
      "sha": "tree123"
    }
  ]
}
```

**优势**：
- 一次请求获取整个仓库的文件树
- 自动递归获取所有子目录
- 包含文件大小、SHA 等元数据

---

### 2. 提交时间缓存

**缓存结构**：
```typescript
Map<string, { date: Date; timestamp: number }>
```

**缓存流程**：
```typescript
1. 检查缓存
   └─ 命中 → 直接返回（无需 API 请求）
   └─ 未命中 → 继续步骤 2

2. 请求 GitHub API
   └─ 成功 → 写入缓存，返回结果
   └─ 失败 → 返回 null
```

**缓存清理**：
- 自动过期：5分钟后删除
- 手动清理：下次请求时检查并删除过期条目

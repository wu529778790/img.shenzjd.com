# 图片统计信息设计

## 背景

用户需要快速了解图床的整体使用情况：有多少图片、占用多少空间、主要是什么格式、最近活跃度如何。目前管理页只有图片列表，缺乏宏观概览。

## 目标

在管理页面顶部添加一组**统计概览卡片**，展示图床核心指标，帮助用户快速掌握存储状况。

## 方案

### 展示位置

放在管理页的搜索栏上方、图片列表上方，作为页面第一屏内容。

```
┌─────────────────────────────────────────────────────────┐
│  图片管理                                      126 张  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│  │ 📷 126   │ │ 📦 48 MB │ │ 🖼️ JPG  │ │ 📅 今天  │  │
│  │ 总图片数  │ │ 总大小   │ │ 占比 62% │ │ 最近上传 │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘  │
│                                                         │
│  [搜索框]  [名称] [大小] [日期]                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  [图片网格/列表]                                  │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 统计项

| 卡片 | 图标 | 指标 | 计算方式 |
|------|------|------|---------|
| 总图片数 | Image | `images.length` | 图片数组长度 |
| 总大小 | HardDrive | `images.reduce(sum)` | 所有 `size` 求和，用 `formatFileSize` |
| 主要格式 | FileImage | 占比最高的格式 | 按扩展名分组，取最大值 |
| 最近上传 | Calendar | 最近的 `uploaded_at` | `Math.max(...dates)`，格式化相对时间 |

### 技术实现

- **纯前端计算**：从 `useImages` 返回的 `images` 数组用 `useMemo` 派生，零 API 改动
- **组件**：新增 `src/components/image/ImageStats.tsx`，接收 `images: ImageFile[]`
- **样式**：shadcn/ui `Card` + `lucide-react` 图标 + `cn` 工具函数
- **相对时间**：用 `date-fns` 的 `formatDistanceToNow`（项目已有 `date-fns` 依赖）

### 组件结构

```
ImageStats (ImageStats.tsx)
├── StatCard × 4
    ├── icon (lucide-react)
    ├── value (大号数字)
    └── label (小号说明文字)
```

### 变更文件

| 文件 | 改动 |
|------|------|
| `src/components/image/ImageStats.tsx` | **新建** — 统计卡片组件 |
| `src/app/management/page.tsx` | 在 `ImageGrid` 上方引入 `<ImageStats images={images} />` |

### 不涉及

- 不改 `useImages` hook（只消费其返回的 `images`）
- 不改 API 层
- 不改 Zustand store
- 不新增图表库（纯文本 + 数字卡片）

## 验收标准

- [ ] 管理页顶部显示 4 张统计卡片
- [ ] 总图片数、总大小、主要格式、最近上传四项数据正确
- [ ] 总大小为 0 时显示"暂无图片"
- [ ] 深色模式下卡片背景和文字颜色正确
- [ ] 无图片时不渲染统计区（或显示空状态）

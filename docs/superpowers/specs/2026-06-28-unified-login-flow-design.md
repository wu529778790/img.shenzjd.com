# 统一登录入口设计

## 问题

当前登录流程存在重复点击的问题：
1. 用户在 `/management` 或 `/settings` 页面看到"需要登录"提示
2. 点击"去登录"按钮跳转到 `/login` 页面
3. 在 `/login` 页面再点击"使用 GitHub 登录"开始认证

用户需要点击两次才能完成登录，体验不佳。

## 目标

消除登录流程中的重复步骤，实现一步登录，统一管理所有需要登录的页面。

## 方案：统一登录入口（方案 C）

### 架构

```
┌─────────────────────────────────────────────────────┐
│                    Next.js App                       │
├─────────────────────────────────────────────────────┤
│                                                       │
│   ┌──────────────┐    ┌──────────────────────────┐  │
│   │  其他页面      │    │       /login              │  │
│   │ (management) │    │   ┌──────────────────┐    │  │
│   │ (settings)   │    │   │ GitHub 登录按钮    │    │  │
│   │ (upload)     │    │   └────────┬─────────┘    │  │
│   └──────┬───────┘    └───────────┼──────────────┘  │
│          │                         │                │
│          │ 未登录时自动跳转         │ signIn('github')│
│          └─────────────────────────┘                │
│                           │                         │
│                    ┌───────▼────────┐               │
│                    │  GitHub OAuth   │               │
│                    └────────────────┘               │
│                                                       │
└─────────────────────────────────────────────────────┘
```

### 组件设计

#### 1. 移除各页面的"去登录"按钮

**需要修改的文件：**
- `src/app/management/page.tsx` - 移除登录提示卡片中的"去登录"按钮
- `src/app/settings/page.tsx` - 移除登录提示卡片中的"去登录"按钮

**修改后行为：**
- 页面显示"需要登录"提示信息
- 移除按钮，仅作为信息展示
- 用户将自动重定向到登录页面

#### 2. 全局未登录拦截

**实现方式：** Next.js Middleware

**文件：** `src/middleware.ts`

```typescript
import { withAuth } from "next-auth/middleware"

export default withAuth({
  pages: {
    signIn: '/login',
  },
})

export const config = {
  matcher: ['/management/:path*', '/settings/:path*']
}
```

**作用：**
- 拦截 `/management` 和 `/settings` 路径下的所有请求
- 未登录用户自动重定向到 `/login`
- 已登录用户正常访问

#### 3. `/login` 页面作为唯一登录入口

**文件：** `src/app/login/page.tsx`

保持不变，继续作为独立的登录页面：
- 显示应用 Logo 和标题
- "使用 GitHub 登录"按钮直接调用 `signIn('github')`

### 错误处理

| 场景 | 处理 |
|------|------|
| GitHub 认证失败 | NextAuth 自动重定向到 `/login?error=...` |
| 网络错误 | 显示错误提示，用户可重试 |
| 用户取消授权 | 返回 `/login` 页面，可重新尝试 |

### 测试场景

1. **未登录访问管理页** → 自动跳转到 `/login`
2. **未登录访问设置页** → 自动跳转到 `/login`
3. **登录成功后** → 重定向回原来访问的页面
4. **直接访问 `/login`** → 正常显示登录页面
5. **已登录访问各页面** → 正常显示内容

### 未来扩展

- 如需支持其他登录方式（Google、GitLab 等），只需在 `/login` 页面添加对应按钮
- 如需保护更多页面，在 middleware config 中添加对应路径

## 💡 为什么不保留提交时间？

### 技术限制

1. **GitHub Contents API**
   - 不返回提交时间
   - 按路径字母顺序排序

2. **Git Trees API**
   - 不返回提交时间
   - 按文件系统顺序返回

3. **GitHub Commits API**
   - 每个文件需要单独请求
   - 50张图片 = 50次请求
   - 100张图片 = 100次请求

4. **GitHub GraphQL API**
   - 不支持批量查询文件提交时间
   - `entries` 字段不接受 `names` 参数
   - `Blob` 类型没有 `commitHistory` 字段

### 结论

- ❌ GitHub 不提供"一次请求获取所有文件+提交时间"的 API
- ✅ 要么接受 1+N 次请求（每个文件请求提交时间）
- ✅ 要么放弃提交时间，只用 1 次请求

**选择方案 C 是最优解**：性能极致、实现简单、稳定可靠。

# ♿ 可访问性声明

ImgX 致力于为所有用户提供无障碍的使用体验，无论他们是否有身体障碍。

## 📋 标准遵循

ImgX 遵循以下可访问性标准：

- ✅ **WCAG 2.1 Level AA** - Web 内容可访问性指南
- ✅ **Section 508** - 美国康复法案第 508 条
- ✅ **EN 301 549** - 欧洲可访问性标准

## 🎯 核心特性

### 键盘导航

所有功能均可通过键盘访问：

- `Tab` - 在交互元素间导航
- `Shift + Tab` - 反向导航
- `Enter` / `Space` - 激活按钮和链接
- `Escape` - 关闭模态框和菜单

### 屏幕阅读器支持

ImgX 已优化对主流屏幕阅读器的支持：

- **VoiceOver** (macOS / iOS)
- **NVDA** (Windows)
- **JAWS** (Windows)
- **TalkBack** (Android)

### 高对比度支持

- 所有文本满足 WCAG AA 对比度要求（≥ 4.5:1）
- 支持系统高对比度模式
- 清晰的焦点指示器

### 动画偏好

- 尊重 `prefers-reduced-motion` 系统设置
- 可暂停或减少动画效果

## 🧪 测试工具

我们使用以下工具确保可访问性：

- **axe DevTools** - 自动化可访问性测试
- **WAVE** - 可视化可访问性反馈
- **Lighthouse** - 综合质量审计
- **手动键盘测试** - 确保完整键盘导航

## 📝 已知问题

如发现任何可访问性问题，请通过以下方式报告：

1. 在 GitHub Issues 中提交，标签为 `a11y`
2. 描述问题场景和复现步骤
3. 说明您使用的辅助技术（如有）

## 🔄 持续改进

可访问性是我们持续改进的领域。我们定期：

- 进行可访问性审计
- 测试主流屏幕阅读器
- 收集团队和用户的反馈
- 更新我们的设计系统
